[COMPARISON: Robinson NSA]
==============================================

SOURCE: Robinson (1961 article, 1966 book "Non-Standard Analysis"); secondary: nLab, HandWiki, Wikipedia

--- The Standard Part Function st() ---

Definition (confirmed via nLab, HandWiki):
  Every FINITE hyperreal x has a unique standard (real) number st(x) such that x - st(x) is infinitesimal.
  Formally: st(x) = x_0 where x_0 is the unique real satisfying |x - x_0| < r for all positive real r.

What st() does:
  - Applies ONLY to finite hyperreals (those not exceeding any real).
  - DISCARDS the infinitesimal residue: st(a + ε) = a for real a and infinitesimal ε.
  - st(ε) = 0 for any infinitesimal ε (confirmed).
  - st(H) is UNDEFINED for infinite H (st() does not collapse infinities to a finite value).

Derivative in NSA (confirmed via LibreTexts Keisler Section 2.1, plan-section-literature.md):
  f'(x) = st( [f(x + ε) - f(x)] / ε )
  where ε is any nonzero infinitesimal.
  Three steps required: (1) substitute infinitesimal, (2) simplify ratio, (3) apply st().

--- Does Robinson Index Infinitesimals? ---

Finding: NO. Confirmed via multiple sources.

From nLab: "Different sequences can generate distinct infinitesimals, but they belong to the same category
without systematic individual labels."

From LibreTexts Keisler 1.5: The product ε·H (infinitesimal × infinite) is explicitly labeled as
INDETERMINATE. "Each of these can be either infinitesimal, finite but not infinitesimal, or infinite,
depending on what ε, δ, H, and K are." Examples:
  - H · (1/H²) = infinitesimal
  - H · (1/H)  = finite (= 1)
  - H² · (1/H) = infinite

From plan-section-literature.md (confirmed by web sources): "NSA has no mechanism to say 'this
infinitesimal came from dividing some quantity by this particular infinity,' so it cannot automatically
recover the product."

--- The NSA Embedding Relationship to IVNA ---

IVNA's embedding (from value-assessment.md, confirmed consistent with NSA literature):
  0_x := x · ε₀    (for a FIXED positive infinitesimal ε₀ in *R)
  ∞_x := x / ε₀
  =;  := st()

Under this embedding:
  0_x · ∞_y = (x · ε₀) · (y / ε₀) = xy   [exact, since ε₀ cancels]
  This IS a valid NSA computation — but only because the specific pairing is tracked.

Robinson's st() discards information: st(ε₀) = 0, throwing away which ε₀ was used.
IVNA's =; collapses: 0_x =; 0, but BEFORE collapsing, the index x is preserved through operations.

Key difference: IVNA tracks provenance explicitly via the index. NSA computes the same arithmetic
but without a mechanism to know WHICH infinitesimal paired with WHICH infinite BEFORE applying st().

--- Information Preservation ---

NSA: NO explicit information preservation mechanism.
  - ε₁ and ε₂ may both be "some infinitesimal" without a canonical label.
  - The product ε · (1/ε) = 1 trivially, but ε · (1/2ε) = 1/2, etc. — you must KNOW which ε
    you're working with to get the right answer.
  - st() called on ε · H without knowing the relationship between ε and H → UNKNOWN result.

IVNA: YES explicit information preservation via indices.
  - 5/0₁ = ∞₅ (the 5 is stored as index).
  - ∞₅ · 0₁ = 5 (recovered exactly).
  - The index IS the provenance tracker that NSA lacks.

--- Claim Verification: "Robinson's st() discards infinitesimal information. IVNA's indices PRESERVE it." ---

VERDICT: ACCURATE. Confirmed by all sources.

st() strips the infinitesimal component and returns only the real part. It has no output channel
for "which infinitesimal produced this result." IVNA's indexed notation provides exactly that channel.
This is the core notational contribution.

SOURCE: LibreTexts Elementary Calculus (Keisler) Sections 1.5 and 2.1, Keisler's book page

--- Derivative Definition (exact, confirmed) ---

"Let f be a real function of one variable. The derivative of f is the new function f' whose value at x
is the slope of f at x. In symbols, f'(x) = st((f(x + Δx) - f(x))/Δx) whenever the slope exists."

"S is said to be the slope of f at a if S = st((f(a + Δx) - f(a))/Δx) for every nonzero infinitesimal Δx."

Key notation: Keisler uses Δx (capital delta, single symbol) for infinitesimal. Not subscripted.

--- Does Keisler Index Infinitesimals? ---

Finding: NO. Confirmed definitively.

"The text does not employ indexed or labeled infinitesimals (such as ε₁, ε₂, etc.). Keisler works
with generic infinitesimals referenced simply as 'Δx' or 'nonzero infinitesimal Δx' without
distinguishing between multiple infinitesimals through indexing."

--- Comparison: Keisler vs IVNA for Derivatives ---

Keisler:  f'(x) = st( [f(x + Δx) - f(x)] / Δx )
IVNA:     f'(x) = [f(x + 0₁) - f(x)] / 0₁    then =;

DIFFERENCE:
1. IVNA uses subscripted 0₁ instead of unsubscripted Δx — the 1 labels which infinitesimal.
2. IVNA's =; (collapse) plays the same role as Keisler's st() — "apply at the end to get a real result."
3. Keisler's st() is defined only for finite hyperreals.
   IVNA's =; collapses 0_x to 0 AND ∞_x to ∞ — it's a broader collapse operator (covers both ends).
4. Mechanically: for standard derivatives, they compute the SAME result via the SAME NSA embedding.
5. The subscript 1 in IVNA is a notational choice (could use any x ≠ 0); Keisler's Δx is similarly
   arbitrary as long as it's infinitesimal.

VERDICT: The derivative computation is THE SAME OPERATION as Keisler's NSA derivative. The notation
differs (indexed 0₁ vs unsubscripted Δx, implicit vs explicit st()). No independent novelty in the
derivative mechanism itself, but the notation is cleaner for presentations where the infinitesimal
index matters (e.g., mixed expressions where two different infinitesimals appear).

[COMPARISON: Sergeyev Grossone]
==============================================

SOURCE: Springer papers, arXiv:1107.5681, Academia grossone computing, PhilPapers, plan-section-literature.md
URL: https://arxiv.org/abs/1107.5681, https://link.springer.com/article/10.1007/s10699-018-9566-y

--- How Grossone Works ---

Grossone ① is defined by the "Infinite Unit Axiom": ① is the number of elements in N (the natural numbers).
It functions as a positional number base. Numbers are written as "grossone numerals" — polynomials in ①
with real coefficients, e.g.:
  - ① (grossone itself — an infinite "natural number")
  - ①/2 (half grossone — "the size of the even numbers")
  - 1/① (an infinitesimal — one over grossone)
  - ①² (grossone squared)

Operations are purely formal (symbolic computation on these polynomials).

--- Grossone Arithmetic for Infinitesimals × Infinities ---

For the specific pair: (1/①) · ① = 1. This holds formally.
For n/① · ① = n. This also holds formally (just cancellation).
But n/① is the ONLY infinitesimal — there is a single canonical infinitesimal 1/①
(and multiples thereof: n/①, n/①², etc.)

Key finding: Sergeyev does NOT have a parameterized family of infinitesimals indexed by real numbers.
The grossone system has:
  {n/①^k : n ∈ R, k ∈ N}
This is a DISCRETE LATTICE (integer powers of 1/①), not a CONTINUOUS real-indexed family.

--- Does Grossone Have a General Product Rule Like 0_x · ∞_y = xy? ---

Finding: NO, not in the general form IVNA uses.

Grossone CAN compute: (n/①) · (m · ①) = nm. This is algebraic cancellation.
But:
  - There is only ONE ① — you cannot have ①_a and ①_b with different labels for unrelated infinities.
  - There is no mechanism where you assign index 5 to an infinity and recover 5 from a later product
    with an unrelated indexed zero.
  - The product (n/①)·(m·①) = nm works because both are explicitly in terms of the same ①.

IVNA's 0_x · ∞_y = xy works for INDEPENDENTLY labeled x and y, even when 0_x came from one
computation and ∞_y from a different computation. Grossone cannot do this.

--- Information Preservation in Grossone ---

n/0 is UNDEFINED in grossone. Sergeyev does not define division by the real zero.
5/0 = ? → NOT addressed. Grossone handles division by ①⁻¹ (giving ①·5 = 5①) but NOT by zero.
So the "can you recover n from n/0=∞→n?" question is simply inapplicable to grossone.

--- Continuous vs Discrete Indexing ---

IVNA: indices are arbitrary real numbers (x ∈ R \ {0}). Continuous family.
Grossone: indices are integers (powers of ①). Discrete lattice.

This is a genuine structural difference. IVNA claims 0_π · ∞_π = π². Grossone has no equivalent
because π is not an integer power of ①.

--- Controversy Status ---

The Gutman-Katz-Kudryk-Kutateladze critique (2016) identified:
1. Circularity in defining ① as "the size of N" while ① ∈ N.
2. No transfer principle → sin(①) is undefined/symbolic.
3. Subsumption by Nelson's IST (Internal Set Theory).
4. Comparison undecidability for general grossone numerals.

IVNA's NSA embedding avoids all of these because IVNA explicitly builds on NSA (which has the
transfer principle) rather than claiming independence from it.

--- VERDICT for Sergeyev Comparison ---

The "indexed product rule" claim is ACCURATE relative to grossone:
  - Grossone's (n/①) · (m·①) = nm is a SPECIFIC case of the IVNA rule 0_n · ∞_m = nm, but
    only when both come from the same ① denominator/numerator.
  - Grossone has no parameterized continuous family {0_x : x ∈ R}.
  - Grossone does not define division by zero at all.
  - The IVNA rule is genuinely more general in structure.

[COMPARISON: Carlström Wheel Algebra]
==============================================

SOURCE: nLab wheel (full axioms), ResearchGate Carlstrom paper, Wikipedia Wheel theory
URLs: https://ncatlab.org/nlab/show/wheel, https://www.researchgate.net/publication/220173535_Wheels_-_on_division_by_zero

--- Wheel Operations (confirmed exact) ---

A wheel extends a commutative ring by adding:
  - A unary reciprocal operation / (not the multiplicative inverse)
  - ∞ = /0 (defined as the reciprocal of 0)
  - ⊥ = 0/0 (the "bottom" absorbing element)

Key axioms (verified from nLab):
  - //x = x (involution)
  - /(xy) = /x · /y
  - 0 · ∞ = ⊥  (CONFIRMED: zero times infinity is the bottom element, NOT a finite number)
  - n · ∞ = ∞  for n ≠ 0, ⊥
  - ∞ + ∞ = ⊥
  - x + ⊥ = ⊥  (⊥ absorbs everything)

--- Does Wheel Algebra Preserve the Numerator in n/0? ---

Finding: NO. Definitively confirmed.

From nLab: "5/0 = 5 · (/0) = 5 · ∞ = ∞"
"The specific numerator (5) is absorbed into the universal ∞. The wheel has a single absorbing
infinity that loses contextual information — n · ∞ = ∞ regardless of which non-zero n you multiply."

Wheel: 5/0 = ∞ (loses 5)
IVNA:  5/0₁ = ∞₅ (preserves 5 as index)

--- Is There Indexing in Wheel Algebra? ---

Finding: NO. There is a SINGLE ∞ element and a SINGLE ⊥ element.
"A single ∞" — confirmed by nLab and Wikipedia.
All divisions n/0 (for n ≠ 0) collapse to the SAME ∞.
There is no ∞_5 or ∞_n in wheel theory.

--- Can You Recover n from n/0 = ∞ in Wheels? ---

Finding: NO. Since all n/0 = ∞ (same element), you cannot distinguish 5/0 from 7/0.
The roundtrip (n/0) · 0 = n FAILS in wheels because 0 · ∞ = ⊥ (not n).

IVNA roundtrip: (5/0₁) · 0₁ = ∞₅ · 0₁ = 5. This works.
Wheel roundtrip: (5/0) · 0 = ∞ · 0 = ⊥. Information lost.

--- What Wheel Algebra DOES Do That IVNA Doesn't ---

1. Handles 0/0 explicitly: 0/0 = ⊥ (a distinct error/absorbing element).
   IVNA: 0_x/0_y would require the "A6 division of virtuals" rule, which is more complex.
2. Full algebraic structure — a proper variety with equational axioms.
3. Every commutative ring embeds into a wheel.
4. Used in computer science for total implementations of division.

--- VERDICT for Carlström Comparison ---

IVNA's claim is ACCURATE: Wheel algebra collapses all divisions by zero to a single ∞,
losing the numerator information. IVNA's indexed approach is genuinely different.

The key contrast:
  Wheel: y/0 = ∞ (single absorbing element, no provenance)
  IVNA:  y/0_x = ∞_{y/x} (indexed result, provenance tracked)

This difference is real, confirmed, and documented.

[COMPARISON: Meadows (Bergstra & Tucker)]
==============================================

SOURCE: arXiv:0901.0823, ResearchGate "Division by Zero in Common Meadows"
URLs: https://arxiv.org/abs/0901.0823, https://arxiv.org/abs/1406.6878

--- Definition ---

A meadow is a commutative ring with a TOTAL multiplicative inverse operator satisfying:
  0⁻¹ = 0  (by definition — the inverse of zero is zero)

Consequence: 5/0 = 5 × 0⁻¹ = 5 × 0 = 0.

"Common Meadows" variant: use ⊥ instead of 0 for 1/0:
  1/0 = ⊥  (absorbing error element, similar to wheel's ⊥)
  5/0 = ⊥

--- Does Meadow Preserve Numerator Information? ---

Finding: NO.

Standard meadow: 5/0 = 0. The numerator 5 is completely lost (result is 0, not even an error marker).
Common meadow: 5/0 = ⊥. The numerator 5 is absorbed into the error element ⊥.

In neither variant does the result carry any information about the original numerator.

--- Comparison to IVNA ---

IVNA:          5/0₁ = ∞₅  (numerator 5 preserved as index)
Meadow:        5/0  = 0   (numerator lost, result is 0)
Common Meadow: 5/0  = ⊥   (numerator lost, result is error)

IVNA's approach is structurally different: it creates a new object ∞₅ that carries the numerator
forward into further computation. Meadows/common meadows terminate the information flow at the
point of division by zero.

--- VERDICT for Meadows Comparison ---

IVNA's claim is ACCURATE. Meadows do not preserve numerator information and do not produce
indexed infinities. They are more algebraically tractable (equational varieties) but sacrifice
information preservation entirely.

--- Surreal Numbers (Conway) ---

Sources: Wikipedia surreal number, Stanford Encyclopedia of Philosophy

Key finding: "In contrast to the real line, to which +∞ and −∞ can be added, there is only one ∞ for ℂ."

Riemann sphere: single unsigned ∞ = compactification point.
  1/0 = ∞ (a single point)

No indexing of infinities. No roundtrip. Purely topological/geometric, not algebraic.

IVNA vs Riemann sphere: Same gap as with wheel algebra — IVNA's ∞ is parameterized, Riemann sphere's is not.

  - The two infinities (±∞) are not indexed by the magnitude of the approaching value.
  - No roundtrip: you cannot recover n from the statement "n/0 approached +∞."

--- SIA / Synthetic Differential Geometry ---

From plan-section-literature.md (confirmed, not re-searched):

  - No division by zero.
  - No indexed infinitesimals or infinities.
  - Derivatives via ε² = 0 axiom (different mechanism from IVNA's indexed approach).
  - Non-classical logic framework (law of excluded middle rejected).

IVNA vs SIA: Different mechanism, different philosophy. SIA is about tangency in smooth toposes.
IVNA is about provenance tracking in arithmetic. Non-overlapping approaches.

--- Numerosity Theory (Benci & Di Nasso) ---

From plan-section-literature.md:
Key finding: Assigns proportional cardinalities (|even numbers| = α/2) using selective ultrafilters.
  - No division by zero.
  - No indexed zeros/infinities.
  - The "α" for |N| is not a canonical real-indexed object.
  - Applications limited to set cardinality — no calculus, no arithmetic of singularities.

IVNA vs Numerosity: IVNA's proportional set sizes (|[0,π]| = ∞_π) implement the same intuition
but as a consequence of the index structure, not as a primary construction.
